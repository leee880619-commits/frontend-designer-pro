# 00 — 맥락 이해 문서 (Context Lock)

> 이 문서는 `frontend-designer-pro` 플러그인 **제작 작업(빌드 타임)**의 합의된 맥락을 고정한다.
> 작성: 2026-06-01 · 모드: ultracode (xhigh + workflow orchestration)

---

## 1. 작업의 두 층위

작업은 반드시 두 층위로 분리해 이해한다. 섞이면 결과물이 어긋난다.

### A층 — 이번 세션이 **만드는 것** (빌드 타임)
6인 프론트엔드 디자인 전문가를 멀티에이전트로 **병렬 소환 → 치열한 토론 → 합의 → 설계 → 구현**하여,
배포 가능한 **Claude Code 플러그인 `frontend-designer-pro`** 를 제작한다.
`/dogfood`(agent-browser)로 실제 산출물이 렌더링·동작하는지 검증한다.
`team_tool/frontend-designer-pro/` 완전 독립 폴더에 만든다 (GitHub·마켓플레이스 연동 전제).

### B층 — 그 플러그인이 **하는 일** (런타임, 팀원이 `/frontend-designer-pro` 호출 시)
1. **질문 먼저 (무조건)** — 아무리 간단한 요청도 건너뛰지 않고 부족한 맥락을 질문으로 정면 돌파.
   md/html 첨부 시 깊게 검토해 의도·활용법 파악. Q&A·맥락 이해 결과를 **결과물 폴더에 md로 만들지
   (경로·관리·생성 여부 포함) 질문 후** 결정.
2. **샘플 N개 제안 여부 질문** — 동의 시 임시/예시 데이터로 레이아웃·디자인 요소를 비교 가능한
   **메인 갤러리 페이지(스냅샷 + 클릭→샘플 링크) + 샘플 페이지들** 생성·제안.
3. **본 작업 수행** — 시안 선택(또는 샘플 생략) 결과에 맞춰 요청 작업 진행.
4. **저장 위치** — 사용자가 질문에 답한 위치 기준.

## 4. 1급 제약 (CRITICAL — 전 과정 적용)

- **비개발자 다수**: 사용자 상당수가 비개발자다. 모든 질문·제안 UI는 **각 방식을 제안한 이유와 추천안을
  근거 있고 쉽게** 설명해야 한다. 전문용어 남발 금지. 토론·기획·구현 전 과정에서 이 점을 1순위로 고려.
- **암묵적 합의 금지**: 모호하면 질문으로 정면 돌파. (CLAUDE.md 협업 규약 동일선상)
- **AI slop 제거**: "AI가 바로 만든 티"를 적극 제거, 사람이 의도적으로 다듬은 수준 지향.
- **자기완결**: 외부 자산 의존 금지.
