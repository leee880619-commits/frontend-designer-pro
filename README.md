# frontend-designer-pro

> **전문가-프론트엔드 디자이너** — `/frontend-designer-pro` 한 번으로, 코드를 몰라도 사람이 의도적으로 다듬은 수준의 프론트엔드 결과물을 받습니다.

---

## 무엇을 하나요

질문 몇 개에 답하면, **그냥 열리는(더블클릭) 완성된 결과물**을 폴더째 만들어 드립니다. 만드는 종류는 네 가지 + 기타입니다.

| 만드는 것 | 쉬운 설명 |
|-----------|-----------|
| **웹앱(화면·페이지)** | 사람들이 클릭하고 입력하며 실제로 '쓰는' 화면. 버튼을 누르면 반응하고, 폼을 채우면 처리됩니다. |
| **공유용 분석 보고서** | 여러 동료가 받아 읽으며 '무슨 일이 일어났는지' 데이터로 이해하는 문서. 차트와 표가 결론을 또렷이 말해줍니다. |
| **C레벨·임원 보고서** | 결정을 내리는 윗분들이 짧게 보고 판단하는 자료. 화려함보다 '믿을 만하다'는 인상과 핵심만 또렷한 구조가 생명입니다. |
| **발표 덱(HTML 슬라이드)** | 회의실 화면에 띄워놓고 말로 설명하는 발표 자료. 뒷자리에서도 읽히고, 한 장에 하나의 메시지만 담습니다. |
| **기타(랜딩·마케팅 등)** | 위 네 가지에 딱 안 맞으면, 무엇을 만들고 싶은지 한 번 더 여쭤 가장 잘 맞는 방식을 찾아드립니다. |

**왜 "전문가급"인가요?** 화면 뒤에서 6명의 전문가 역할(아트디렉터, 디자인시스템 설계자, 색채과학자, 접근성/사용성 리드, 데이터시각화 전문가, 프론트엔드 구현 엔지니어)의 점검표가 순서대로 돌아갑니다. 색 대비·글꼴 짝·여백 위계·차트 정직성·색맹 동료 배려까지 자동으로 챙기고, 내보내기 전에 "잘 읽히는지 / AI가 바로 만든 티가 안 나는지 / 색맹 동료도 구분되는지"를 전수 점검한 뒤 문제가 있으면 알아서 고쳐 완성합니다. **당신이 숙제를 떠안지 않습니다.**

## 설치법

### 방법 A — 마켓플레이스로 설치 (권장)

Claude Code 안에서 아래 두 줄을 입력합니다.

```
/plugin marketplace add https://github.com/leee880619-commits/frontend-designer-pro
/plugin install frontend-designer-pro
```

- 첫 줄: 이 플러그인이 들어 있는 "장터(마켓플레이스)"를 등록합니다.
- 둘째 줄: 장터에서 `frontend-designer-pro`를 골라 설치합니다.

### 방법 B — GitHub 저장소를 직접 연동

이미 사내 GitHub에 올려둔 저장소를 연동하려면 같은 주소를 씁니다.

```
/plugin marketplace add https://github.com/leee880619-commits/frontend-designer-pro
/plugin install frontend-designer-pro
```

> **연동 주소 안내**: 위 주소(`homepage`)는 `.claude-plugin/plugin.json`에 적혀 있는 값입니다. 본인 계정이나 다른 저장소에 올렸다면, 저장소에 `push`한 뒤 `plugin.json`과 `marketplace.json`의 `homepage`·`repository` 값을 자신의 실제 주소로 바꿔 주세요. (코드를 몰라도 됩니다 — 텍스트 편집기로 그 URL 한 줄만 바꾸면 됩니다.)

설치가 끝나면 Claude Code에서 `/frontend-designer-pro`가 자동완성 목록에 나타납니다.

## 사용법

1. **호출** — Claude Code에 `/frontend-designer-pro` 라고 입력하거나, "임원 보고서 만들어줘" 처럼 자연어로 말합니다. 첨부 파일(md·html)이 있으면 먼저 꼼꼼히 읽어 질문 수를 줄여 드립니다.
2. **질문에 답하기** — 딱 한 번, 꼭 필요한 것만 **그림과 객관식**으로 묻습니다. 각 보기에는 ★추천 디폴트(기본 선택)가 미리 켜져 있으니, **"알아서 해주세요"를 골라도, 안 골라도 전문가 기본값으로 최고 품질**이 나옵니다. (색·접근성·반응형 같은 품질 하한선은 묻지 않고 항상 자동 적용합니다.)
3. **(선택) 샘플 비교** — "서로 다른 분위기 시안 3개를 나란히 비교해 보시겠어요?"에 동의하면, 예시 데이터로 채운 시안들을 한 화면에서 그림으로 비교하고 마음에 드는 하나를 고릅니다. 건너뛰어도 됩니다.
4. **결과 폴더 저장** — 당신이 답한 위치에 사람이 알아볼 이름으로 저장됩니다. **`index.html`을 더블클릭하면 브라우저에서 바로 열립니다.** 터미널·서버·빌드 과정이 전혀 필요 없습니다.

처음부터 끝까지 한 화면 흐름 안에서 진행되며, 중간에 멈춰도 다음에 이어서 시작할 수 있습니다.

## 자기완결 보장 — 다른 것 안 깔아도 됩니다

이 플러그인은 **혼자서 완전히 동작**합니다. `frontend-design`, `frontend-designer`, `color-expert` 같은 다른 디자인 스킬이 **설치돼 있지 않아도** 결과물 품질이 그대로입니다. 필요한 전문 지식(미적 방향 갤러리, 색·토큰 규칙, 접근성 점검표, 차트 선택 기준, 무빌드 골격, 비개발자 카피 가이드)을 모두 플러그인 안 `references/` 폴더에 텍스트로 담아 두었기 때문입니다.

산출물이 약속한 품질("잘 읽히는지 / 색맹 동료도 구분되는지")은 **말이 아니라 실제로** 점검합니다 — 더블클릭만으로 도는 점검 도구(`tools/_audit.html`)가 대비·색맹·토큰을 한 화면에서 실측하고, 지식 묶음의 핵심 값이 시간이 지나도 어긋나지 않도록 일관성 검사(`tools/kb-consistency-check.sh`)가 지킵니다. 두 도구는 **만드는 사람이 검증할 때만** 쓰며, 받으시는 결과물 폴더에는 들어가지 않습니다.

또한 결과물은 **CDN(외부 서버)이 막혀 있어도** 한글이 깨지지 않도록 폰트 폴백을 갖추고, 구형 브라우저나 Confluence 임베드 환경에서도 색이 안전하게 표시되도록 설계됩니다.

## 비개발자라면 — 이것만 알면 됩니다

- **터미널을 몰라도 됩니다.** Claude Code 채팅창에 `/frontend-designer-pro`만 입력하면 시작합니다. 결과물은 파일 탐색기에서 **`index.html`을 더블클릭**하면 열립니다.
- **한 곳만 바꾸면 전체가 바뀝니다.** 색이나 글꼴을 나중에 바꾸고 싶으면, 결과 폴더의 `tokens.css` 파일 **한 곳**만 고치면 됩니다. 어디를 바꾸는지는 함께 들어 있는 **편집 가이드 1장**이 한국어로 안내합니다. (예: "브랜드 색을 바꾸려면 이 줄의 색만 바꾸세요.")
- **결과는 신호등으로 알려 드립니다.** 어렵게 숫자로 말하지 않고 🟢"잘 읽힘" / 🟡"큰 글씨만 OK" / 🔴"자동으로 고쳤음" + 실생활 한 줄로 보고합니다.
- **문제가 있으면 대신 고칩니다.** 색이 흐려 안 읽히거나 그림 설명이 빠진 경우, 숙제를 떠넘기지 않고 자동으로 보정한 뒤 완성합니다.

## 결과물 폴더 구조

저장된 폴더는 사람이 알아볼 이름으로 정리됩니다 (`dist/`·`.next/` 같은 빌드 찌꺼기는 들어가지 않습니다).

```
내-결과물/
├── index.html          ← 여기를 더블클릭하면 열립니다
├── tokens.css          ← 색·글꼴이 한곳에 (여기만 바꾸면 전체가 바뀜)
├── assets/             ← 폰트·이미지 등 부속 파일
├── samples/            ← (선택) 비교했던 시안들
├── 편집-가이드.md       ← "이 숫자를 바꾸세요" 1장 안내
├── 검증-요약.md         ← "브라우저 4종 깨짐 없음, 2초 내 로딩, 오류 0건" 식 보고
└── a11y-report.md      ← 접근성(잘 읽히는지) 점검 결과
```

### 플러그인 자체의 폴더 구조 (참고)

```
frontend-designer-pro/
├── .claude-plugin/
│   ├── plugin.json              ← 플러그인 정보(이름·버전·연동 주소)
│   └── marketplace.json         ← 장터 등록 정보
├── skills/
│   └── frontend-designer-pro/
│       └── SKILL.md             ← 전체 워크플로우 본체(질문→샘플→산출→저장)
├── agents/
│   ├── design-lead-pro.md       ← 본 작업 산출 담당
│   └── design-redteam-pro.md    ← 내보내기 전 무결성 감사 담당
├── references/                  ← 자기완결 지식 번들 (외부 스킬 없이 동작)
│   ├── _block-card.md           ← [항상 참조] 하드 게이트 1페이지(무엇이 BLOCK인지)
│   ├── _contract.md             ← [항상 참조] 상수·대비 임계·토큰명 단일 기준
│   ├── tokens.contract.json     ←  └ 위 계약의 기계 판독용 미러(값 1벌만 유지)
│   ├── aesthetic-directions.md  ← 미적 방향 갤러리(그림으로 고르기)
│   ├── anti-slop-checklist.md   ← AI 티 제거 점검표
│   ├── art-brief-template.md    ← 디자인 방향 1페이지(모든 결정의 기준)
│   ├── color-token-contract.md  ← 색 규칙(OKLCH·대비 게이트)
│   ├── token-system.md          ← 색·글꼴 토큰 체계(한곳 수정 구조)
│   ├── a11y-checklist.md        ← 접근성 점검표
│   ├── chart-decision.md        ← 차트 선택·정직성 규칙
│   ├── build-boilerplate.md     ← 무빌드 더블클릭 골격
│   └── non-dev-copy.md          ← 비개발자 카피 가이드
│                                  (계약 2종은 항상, 깊은 지식 9종은 필요할 때만 펴봅니다)
├── tools/                       ← 산출물 자가 점검 도구 (검증용 — 결과물 폴더 미포함)
│   ├── _audit.html              ← 더블클릭 무빌드 감사기(대비·토큰·색맹을 한 화면에서 실측)
│   └── kb-consistency-check.sh  ← 지식 묶음 상수 일관성 검사(시간이 지나도 안 어긋나게)
├── examples/                    ← 데모 샘플 갤러리(예시 데이터로 산출 수준 미리보기)
├── docs/                        ← 설계 근거 문서(유지보수 참고용)
├── README.md                    ← 이 문서
└── LICENSE                      ← MIT
```
